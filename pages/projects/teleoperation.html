Featured Project

Teleoperation

Real-time kinematic mirroring between a custom miniature arm and a UR5e industrial manipulator over ROS2.

Role Solo engineering project
Status Ongoing
Stack Python, C++, ROS2, ESP32

Overview

This project builds a live teleoperation link between a custom-fabricated miniature robotic arm and a Universal Robots UR5e industrial manipulator. An operator moves the small, lightweight leader arm by hand and the UR5e mirrors every joint angle in real time — translating human-scale intuitive motion into precise industrial manipulation without direct physical contact with the larger robot.

The system is designed as a solo engineering project, covering hardware fabrication, embedded firmware, and a full ROS2 software stack. It is still actively being developed, with force-feedback and compliance control as the next milestones.

System Architecture

  • Leader arm (custom): A 6-DOF desktop arm built from 3D-printed links and low-cost servo motors. Potentiometers at each joint provide continuous angle feedback, read by an ESP32 microcontroller at 100 Hz over I2C.
  • ESP32 firmware (C++): Reads raw ADC values, applies a calibration map and low-pass filter, then publishes normalised joint angles to a ROS2 topic via micro-ROS over serial USB.
  • ROS2 bridge (Python): Subscribes to the joint-angle topic, applies the kinematic scaling transform between the two arms' link-length ratios, and publishes scaled target positions to the UR5e driver node. A jitter-compensation buffer smooths high-frequency noise before commands are dispatched.
  • UR5e driver: Uses the official ur_robot_driver ROS2 package with a real-time control interface. The bridge feeds scaled joint targets at the robot's control frequency, keeping motion smooth and latency below perceptual threshold.

Key Technical Challenges

  • Kinematic scaling: The leader and follower arms have different link lengths, so a direct joint-angle copy produces incorrect Cartesian trajectories. A per-joint scale factor derived from forward-kinematics analysis corrects the mapping.
  • Latency and jitter: USB serial introduces variable latency. A sliding-window median filter and a command-dispatch throttle keep the UR5e motion smooth without introducing perceptible lag from the operator's perspective.
  • Workspace limits: The UR5e's joint limits differ from the leader arm. Soft limit clamping on the bridge node prevents the follower from ever receiving an out-of-range command, avoiding protective stops.
