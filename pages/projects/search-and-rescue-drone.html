Project

Search and Rescue Drone

Multi-robot navigation and coordination in simulation.

Role Robotic System Programming
Year 2024
Stack C++, Webots, LiDAR

Overview

This project simulates a multi-robot search-and-rescue mission inside the Webots robotics simulator. Two autonomous C++ agents navigate a procedurally generated maze, cooperate to locate a survivor zone, and return home — without any human intervention or pre-loaded map.

The scenario is modelled on real SAR operations where a team of small drones is deployed into a collapsed structure: each robot must independently explore unknown territory, share critical findings over a limited communication channel, and execute a coordinated extraction sequence.

How It Works

  • Wall-following navigation: Each robot maintains a 0.2–0.5 m lateral clearance from the nearest wall using distance sensor feedback and a PID heading controller. The algorithm handles both left-hand and right-hand rule variants to improve maze coverage.
  • Target zone detection: A zone is considered a survivor location when the robot simultaneously reads walls on its front, left, and right sensors. The detection logic filters transient false positives with a multi-sample confirmation window.
  • Inter-robot communication: Upon confirming a target zone, the discovering robot broadcasts a custom TARGET_FOUND message over a shared virtual channel. The partner robot acknowledges receipt, logs the discovery timestamp, and transitions to its homing state.
  • Synchronised homing: Both robots retrace a logged sequence of waypoints back to their start positions, achieving return accuracy of ±1 m. Odometry drift is corrected with periodic wall-contact re-calibration.
  • Continuous data logging: Throughout the mission, each robot writes timestamped wall-clearance distances and heading data to a CSV log, enabling full post-mission trajectory reconstruction and analysis.
