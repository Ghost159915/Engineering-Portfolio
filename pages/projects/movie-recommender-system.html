AI/ML Project

Movie Recommender System

Recommender system that learns user preferences from rating history and ranks unseen films by predicted relevance.

Focus Collaborative Filtering
Stack Python, NumPy, scikit-learn
Status In progress

Overview

The Movie Recommender System is a Python project that explores the core techniques behind modern recommendation engines. Given a user's historical film ratings, the system predicts which unrated films that user would enjoy most and returns a ranked shortlist. The project is built for learning and comparison: multiple algorithms are implemented from scratch or with minimal library assistance, then evaluated head-to-head on the MovieLens dataset.

Approaches Compared

  • User-based collaborative filtering: Finds users with similar rating histories using cosine similarity, then aggregates their ratings — weighted by similarity score — to predict the target user's preferences. Simple and interpretable, but struggles at scale as the user matrix grows.
  • Item-based collaborative filtering: Computes item–item similarity from the transposed rating matrix. More stable than user-based filtering because item relationships change slowly over time; also easier to pre-compute and cache.
  • Matrix factorisation (SVD): Decomposes the sparse user–item rating matrix into latent user and item factor vectors. Handles sparsity better than neighbourhood methods and generalises to users with fewer ratings. Implemented using truncated SVD via NumPy and compared against the surprise library's SVD baseline.

Evaluation

All models are evaluated on a held-out test split using RMSE (rating prediction accuracy) and Precision@K / Recall@K (ranking quality for top-N recommendations). A popularity baseline — always recommending the globally highest-rated films — anchors the comparison and highlights the value of personalisation.

Status & Next Steps

Memory-based and SVD models are complete and benchmarked. The next phase introduces a neural collaborative filtering model (embedding layers trained with PyTorch) and explores content-based signals — genre, director, cast — to handle the cold-start problem for new users with no rating history.
