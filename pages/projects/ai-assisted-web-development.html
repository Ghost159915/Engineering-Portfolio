AI Assisted Web Development

Role Gen AI Assisted Programming
Year 2025
Stack HTML, CSS, JavaScript

Overview

This project is the portfolio site you're reading right now. It was designed and built using AI as an active development partner — not just for boilerplate, but for architectural decisions, layout iteration, component design, and copy review. The goal was to ship a polished, production-quality site rapidly while genuinely learning the web stack in the process.

Rather than using a site builder or template, every line of HTML, CSS, and JavaScript was written from scratch via a conversational prompting workflow with Claude Code. This forced a deep engagement with semantic structure, responsive design, and browser behaviour that a template would have hidden.

Key screenshot

How it was built

  1. Architecture first: Defined the page hierarchy (About, Projects, Contact, per-project detail pages) and component structure before writing any code, using AI to stress-test the information architecture against real user flows.
  2. Prompt-driven HTML/CSS: Each section was built through iterative prompts — describing the desired layout, receiving a candidate implementation, reviewing it against design goals, and refining. This cycle exposed the reasoning behind every CSS rule.
  3. Responsive layout system: The grid and flex layout system uses CSS custom properties for spacing and breakpoints, making the design consistent and easy to extend as new project pages are added.
  4. Progressive enhancement: Interactive components (the featured carousel, the contact form AJAX handler) were added after static structure and styling were stable, keeping the core content accessible without JavaScript.
  5. Version control and deployment: All changes were committed incrementally to GitHub with meaningful commit messages. Deployed to Netlify with a custom domain and HTTPS via automatic Let's Encrypt provisioning.

The outcome demonstrates that AI-assisted development is not about writing less code — it is about spending more time on decisions and less time on syntax lookups. Every snippet was reviewed, understood, and adjusted before committing.
